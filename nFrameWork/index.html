Plug-in

nFrameWork.js는 테그에 class=""와 data-np-foo="" 속성 추가로 간단하게 기능을 구현하는 플러그인입니다.

웹 접근성을 고려하여 설계되어있어 추가적인 접근성 보완을 필요로 하지 않습니다.

WAI-ARIA 속성이 자동완성 됩니다.

단독으로 nFrameWork.js만 사용할 수 있습니다.

MS의 경우 IE9+ 부터 지원합니다.

tool tip

툴팁을 사용하기 위해서 우선 <body>의 직계 자손을 선언해야 합니다.

<body>의 직계 자손이라면 몇 번째 인지는 상관없으며 단 한번만 선언하면 됩니다.

클레스 네임에 테마를 추가하여 색상을 변경 할 수도 있습니다.

<body>
	<span id="tooltip"></span>
</body>

이후 툴팁이 표시되길 원하는 테그에 class="tooltip-t, r, b, l"와 data-np-tooltip='content' 속성을 설정하니다.

t, r, b, l은 툴팁이 표시될 방향을 의미합니다.

<span class="tooltip-b" data-np-tooltip="show to bottom">into</span><br>
<span class="tooltip-l" data-np-tooltip="show to left">into</span><br>

layer popup

레이어 팝업은 부트스트랩의 모달과 동일한 목적과 기능을 가집니다.

팝업을 실행시킬 테그에 data-np-popup="foo" 속성을 추가합니다.

여기서 속성값은 팝업창의 id=""값을 가리킵니다.

  • 코드
  • 결과
<input data-np-popup="foo21" type="button" value="Popup Button!">

다음으로 팝업이 실행될 시 표시될 테그에 지정한 id=""값과 class="popup-box"값을 입력합니다.

팝업 내부에 팝업을 닫는 테그를 설정하고 싶을 경우 해당 테그에 data-np-popup=""과 같이 빈 문자열의 속성을 추가합니다.

  • 코드
  • 결과
<div id="foo21" class="popup-box">
	content<br>
	<input data-np-popup="" type="button" value="Close Popup Button!">
</div>

tab panel

텝 페널은 제한된 공간에 여러 컨텐츠를 제공해야 할 경우 유용하게 사용할 수 있습니다.

우선 텝페널을 감쌀 테그를 class="tabwrap"속성을 포함하여 생성합니다.

이후 내부에 텝 헤더부분을 <ul><li> 테그를 사용하여 생성합니다.

이때 <li>는 class="tabhead" 속성을 포함하여야 합니다.

마지막으로 텝 헤더에 해당하는 페널부분을 class="tabbody"속성을 포함하여 생성합니다.

class="tabwrap"에 테마 클레스 네임을 추가하여 텝 헤더 부분의 비활성화 상태일 때의 색상을 변경 할 수도 있습니다.

slide

<div class="slide-box">와 같이 클레스 네임을 할당하여 슬라이드 영역을 선언합니다.

슬라이드 자동전환(재생) 시간의 기본값은 4000ms 입니다.

전환 시간을 조절하고 싶을 경우 슬라이드 영역 테그에 data-np-slide-speed="*" 속성과 값을 추가합니다.

여기서 *는 희망하는 밀리초(ms)단위의 시간을 말합니다.

다음으로 슬라이드 영역 내부에 <ul class="slide-item">과 같이 클레스 네임을 할당하여 컨텐츠 영역을 선언하고, 그 내부에 컨텐츠 수량만큼의 <li>를 선언하고 컨텐츠를 삽입합니다.

기본적으로 슬라이드 자동전환(재생)은 정지상태 입니다.

자동재생 상태를 희망할 경우 컨텐츠 영역에 class="played"값을 추가로 할당해야 합니다.

슬라이드의 기본 크기는 width: 100%에 16:9 비율로 설정되어 있으며 문서 크기에 따라 자동 조절 됩니다.

슬라이더의 크기를 설정해야 할 경우 css를 이용하여 <div class="slide-box">셀렉터에 max-width, max-height를 % 또는 px 단위로 통일하여 값으로 할당해주어야 합니다.

이전, 다음 버튼이 필요할 경우 슬라이드 영역 내부에 <ul class="slide-btn-prne">와 같이 이전, 다음 버튼영역을 선언하고 내부에 비어있는 <li> 두개를 선언합니다. (선택)

리스트 바로가기 버튼이 필요할 경우 슬라이드 영역 내부에 <ul class="slide-btn-goto">와 같이 리스트 바로가기 버튼영역을 선언하고 내부에 비어있는 <li>를 컨텐츠 수량만큼 선언합니다. (선택)

슬라이드 재생, 정지 버튼이 필요할 경우 <div class="slide-btn-play">와 같이 선언합니다. (선택)

    일반 사용자 UX

    슬라이드가 자동재생 상태이더라도 컨텐츠 위에 마우스가 위치할 경우 마우스가 컨텐츠에서 이탈할 때까지 일시정지 상태가 됩니다.

    이러한 이유는 사용자가 집중하고 있는 컨텐츠가 자동재생에 의한 몰입방해로부터 보호하기 위함입니다.

    키보드 사용자

    불필요하고 과도한 포커스 순서를 피하기 위해 컨트롤러 버튼에 포커스 할 수 없게 설정되어 있습니다.

    대신 해당하는 컨텐츠 또는 컨텐츠 내부의 포커스 가능한 요소가 포커스 되어있을 경우 해당 컨텐츠로 화면이 전환됩니다.

    이후 포커스 된 해당 요소가 포커스를 잃을때까지 자동재생 상태이더라도 화면전환은 이루어지지 않습니다.

    스크린 리더

    슬라이드를 <ul>, <li>로 구성된 컨텐츠로 인식합니다.

    따라서 스크린리더에 의존하는 사용자가 컨트롤 버튼을 인식하여 혼란스러워 하지 않도록 하기위해 컨트롤 버튼은 스크린리더가 인식하지 못하도록 설정되어 있습니다.

    nFrameWork.js에 내장되어 있는 slide는 기본기능만 지원합니다.

    자바스크립트 API를 통해 슬라이드의 고급기능을 사용해야 할 경우 nSlider.js라는 별도의 슬라이드 플러그인을 사용해야 합니다.

    spy

    spy는 현재 문서의 id="foo" 값을 추적하여 해당 테그영역이 화면에 노출되는 동안만 <a href="#foo"> 테그에 class="spied"값을 부여합니다.

    즉, 부트스트랩의 스크롤스파이와 동일한 기능을 구현합니다.

    <ul id="spy-box">와 같이 아이디를 할당하여 스파이 영역을 선언합니다.

    스파이 영역은 문서에 한번만 선언할 수 있습니다.

    링크를 클릭하여 화면에 노출될 때 스크린 상단으로부터 오프셋 보정이 필요할 경우 data-np-spyoffset="*px"속성과 값을 추가합니다.

    여기서 *은 실수값 입니다.

    다음으로 영역 내에 <li> <a href="#foo">를 선언합니다.

    <a>의 href=""속성 양식이 #이 아니거나 문서 내 해당 값의 셀렉터를 찾을 수 없는 경우 해당 링크는 무시합니다.

    코드 
    <ul id="spy-box" data-np-spyoffset="32px">
	<li><a href="#foo1">link</a></li>
	<li><a href="#foo2">link</a></li>
	<li><a href="#foo3">link</a></li>
</ul>
    결과 이 문서의 우측 네비게이션 참고

    Icons

    nIcons.css는 클레스네임 추가로 간단하게 사용가능한 아이콘을 제공합니다.

    아이콘의 크기는 폰트사이즈를 상속받으며 크기를 변경하더라도 svg 형식처럼 해상도 깨짐현상이 없습니다.

    웹 접근성을 고려하여 아이콘 설명이 필요할 경우 해당 테그 내에 font-size: 0속성을 가진 테그를 추가하여 텍스트 삽입하는 것을 권장합니다.

    WAI-ARIA 또는 title="" 속성을 사용해도 되지만 스크린 리더에 따라 해당 구문을 읽지 못하는 경우가 있기 때문입니다.

    색상 변경은 지원되지 않으므로 필요시 커스터마이징 해야 합니다.
    nIcons.css는 #000000 색상값만 사용되었습니다.

    MS의 경우 IE9+ 부터 지원합니다.
